# JB Financial Toolings

A personal finance manager with a built-in, double-encrypted password vault. Single portable encrypted file, works offline, syncs over any file-syncing service (Dropbox, Google Drive, etc.) you point it at.

Two apps share one file format:

- **`finance_app/`** — desktop app (Python + PySide6/Qt). Full feature set: expenses, debts, investments, FDs, borrowings, multi-currency, dashboard/reporting, and the password vault.
- **`android/`** — Android companion app (Kotlin). Vault (read/write) + Dashboard (read-only) for now, syncing the same file via the Google Drive app (or any Storage Access Framework provider) — no cloud API keys involved.

See [`docs/INSTALLATION.md`](docs/INSTALLATION.md) for step-by-step install/build instructions for macOS and Android, [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) for how the encryption, data model, and cross-app compatibility work, and [`jb_finacial_tooling_plan.md`](jb_finacial_tooling_plan.md) for the original project plan and phased build order.

## Features

- Expense tracking with recurring transactions, categories, filters
- Debt/loan payoff planning — snowball vs. avalanche, projected payoff dates and interest cost
- Investments (stocks/funds), fixed deposits, other holdings — manual pricing, gain/loss, portfolio rollups
- Borrowings — money lent/owed, due dates, status
- Multi-currency with a manual exchange rate table and base-currency conversion
- Dashboard — net worth over time, expense breakdown, debt payoff timeline, investment allocation
- Password vault (logins, secure notes, cards, identities) with its own independent encryption layer, clipboard auto-clear, and app auto-lock

## Security model

- Master password → **Argon2id** → root key
- Root key → **HKDF** → two independent subkeys: `finance_key` (encrypts the whole file) and `vault_key` (vault items only)
- File body: **Fernet** (AES-128-CBC + HMAC-SHA256)
- Vault item payloads: **AES-256-GCM**, one random nonce per item, authenticated (tamper-evident)

Full detail, including the exact on-disk format both apps read/write, is in [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

## Desktop app (`finance_app/`)

### Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r finance_app/requirements.txt
```

### Run

```bash
source .venv/bin/activate
python -m finance_app.main
```

First run: pick a location for the data file (defaults to `~/finance_data.enc`) and set a master password. Later runs unlock the same file with that password. The file is a portable single file — put it in a Dropbox/Drive-synced folder to sync across desktops.

### Tests

The desktop app doesn't have an automated test suite yet; verification during development has been via ad hoc scripts exercising `finance_app.data`/`finance_app.services` and a headless (`QT_QPA_PLATFORM=offscreen`) Qt smoke test of the full UI.

## Android app (`android/`)

Two Gradle modules:

- **`crypto/`** — pure Kotlin/JVM port of the encryption logic, no Android dependency. Has its own JUnit test suite that verifies it against real fixtures generated by the Python app.
- **`app/`** — the Android application (Jetpack Compose).

### Setup

Requires Android Studio (bundles the JDK, Android SDK, and Gradle plugin tooling used here). Open the `android/` directory as the project root.

### Build & test from the command line

```bash
cd android
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"  # or your JDK 17+/21
./gradlew :crypto:test              # crypto module vs. real Python-produced fixtures
./gradlew :app:testDebugUnitTest    # FinanceRepository vs. a real Python-produced .enc file (Robolectric)
./gradlew :app:assembleDebug        # produces app/build/outputs/apk/debug/app-debug.apk
```

### Run

Install the debug APK on a device or emulator (`adb install -r app/build/outputs/apk/debug/app-debug.apk`), launch the app, pick the `.enc` file created by the desktop app (from local storage or from the Google Drive app), and enter the master password.

Auto-lock triggers whenever the app leaves the foreground.

## Regenerating cross-language test fixtures

The Kotlin tests decrypt real files produced by the Python app rather than synthetic Kotlin-only data. If the on-disk format or Python crypto code changes, regenerate:

- `android/crypto/src/test/resources/crypto_fixture.properties` — raw key/token/ciphertext values, generated by a short Python script using `finance_app.security.crypto` / `vault_crypto` directly (see commit history for the generator script).
- `android/app/src/test/resources/repository_fixture.enc` — a full encrypted data file with sample vault items and a currency/dashboard snapshot, generated via `finance_app.data.database.DatabaseManager` + the relevant services.
